# 写作范式参考指南

本文档提供常见内容创作范式的参考，供 AI 在分析样张时识别和提取。

---

## 一、核心心法：创造概念

优秀文章的本质是**创造可传播的概念**，而非堆砌观点。

### 概念创造方法

| 方法 | 说明 | 示例 |
|------|------|------|
| **公式化** | 把洞察变成公式 | 产品价值 = 能力 × 信任 × 频率 |
| **对仗化** | "不是 A，而是 B" | 不是输出观点，而是创造概念 |
| **命名化** | 给现象起名字 | 「完成困难症」|
| **转向化** | 从 A 到 B 的转移 | Prompt Engineering → Context Engineering |

## 二、常见结构范式

### 2.1 多层推演结构

适用于深度分析类文章。

```
开头 → 现象/案例 → 初步洞察 → 自我质疑 → 深层推演 → 结尾
```

**关键句式**：
- "好吧，[初步结论]"
- "但，这真的对吗？"
- "后来我开始怀疑这个结论"
- "真正的 [X]，可能不是 [A]，而是 [B]"

### 2.2 对比验证结构

适用于产品分析、方案对比。

```
背景 → Before(现状) → After(新方案) → 核心差异 → 结论
```

**关键句式**：
- "那么问题来了"
- "相比之下"
- "核心差异在于"

### 2.3 案例分析结构

适用于经验分享、复盘总结。

```
背景 → 冲突/问题 → 解决过程 → 结果 → 方法论提炼
```

**关键句式**：
- "当时的处境是"
- "我当时的做法是"
- "回头看，我总结了几点"

## 三、语言风格特征库

### 3.1 句式偏好

| 特征 | 识别标志 | 示例 |
|------|----------|------|
| **短句为主** | 平均 < 25 字/句 | "震惊。麻了。确实。" |
| **中等长度** | 平均 25-50 字/句 | 正常叙述句式 |
| **长短结合** | 有意识的节奏变化 | 短句 + 长句交替 |

### 3.2 标点符号风格

| 符号 | 使用效果 | 示例 |
|------|----------|------|
| **…** | 思考停顿、欲言又止 | "坦白讲，当时的感觉就是…一般般吧" |
| **（）** | 情绪吐槽、补充说明 | "（这很可怕）" |
| **「」** | 概念封装、术语强调 | 「Product-Me Fit」|

### 3.3 常见过渡词

- **引入**：坦白讲、说实话、说白了、简单来说
- **转折**：但、不过、然而、却、反而
- **递进**：更重要的是、本质上、说到底
- **举例**：比如、像、举个例子

## 四、思考模式识别

### 4.1 多层推演

特征：
- 有明确的"第一层""第二层"标记或暗示
- 使用自我质疑句式
- 从表象逐步深入到本质

识别标志：
```
第一层：现象描述
    ↓
第二层：初步结论（但存疑）
    ↓
第三层：深层洞察（本质）
```

### 4.2 情感升华

特征：
- 从理性分析转向感性共鸣
- 使用个人经历或隐喻
- 结尾引发共鸣而非总结

常见主题：
- 成长焦虑 → 扎根隐喻
- 技术变革 → 人文关怀
- 失败经历 →  resilience

### 4.3 反直觉洞察

特征：
- 先抛出常识观点
- 用"但""然而"转折
- 提供意想不到的角度

关键句式：
- "很多人都认为…但事实是"
- "表面看是 A，本质是 B"
- "这个判断太简单了，因为…"

## 五、开头与结尾套路

### 5.1 开头套路

| 类型 | 特征 | 适用场景 |
|------|------|----------|
| **自我介绍** | Hi，我是… | 个人随笔、经验分享 |
| **场景引入** | 描述一个具体场景 | 故事性文章 |
| **提问开场** | 抛出一个问题 | 引发思考的文章 |
| **反差开场** | "我曾经…但现在…" | 认知转变类 |
| **金句开场** | 一句有力的观点 | 观点鲜明文章 |

### 5.2 结尾套路

| 类型 | 特征 | 适用场景 |
|------|------|----------|
| **开放问题** | 以?结尾，邀请思考 | 启发性文章 |
| **观点收束** | 总结核心洞察 | 分析类文章 |
| **行动号召** | 建议读者做什么 | 方法类文章 |
| **情感共鸣** | 引发情绪共振 | 个人叙事 |
| **自我披露** | 关于我/关注我 | 个人品牌 |

## 六、质量检查要点

### 内容层面
- [ ] 是否有原创概念/公式？
- [ ] 是否有案例支撑？
- [ ] 是否有独特洞察（非人云亦云）？
- [ ] 是否有情感内核？

### 结构层面
- [ ] 结构是否清晰？
- [ ] 是否有明确的起承转合？
- [ ] 段落长度是否有节奏感？

### 语言层面
- [ ] 语言风格是否一致？
- [ ] 是否有个人特色表达？
- [ ] 是否避免了 AI 腔？

---

## 七、模板提取维度速查

当分析样张文章时，从以下维度提取特征：

1. **结构维度**
   - 整体组织方式
   - 段落长度分布
   - 章节划分逻辑

2. **语言维度**
   - 句子平均长度
   - 标点符号偏好
   - 过渡词使用
   - 独特表达句式

3. **思考维度**
   - 论证逻辑类型
   - 深度层次数
   - 转折方式

4. **风格维度**
   - 开头套路
   - 结尾套路
   - 情感基调
