# WPS Note 智能搜索 - 工作流架构

> 本文档指导 Claude 如何使用 MCP 工具完成智能搜索任务。
> 这不是代码框架，而是工作流指导。

---

## 1. 核心工作流

```
用户查询
    ↓
[意图解析] ──→ 提取实体、识别维度、判断搜索模式
    ↓
[策略执行] ──→ 选择工具链、分层调用、结果融合
    ↓
[资产沉淀] ──→ 保存搜索历史、建立隐性关联
    ↓
返回结果
```

---

## 2. 意图解析层

### 2.1 解析目标

从自然语言查询中提取：
- **实体**：人名、项目名称、时间表达式
- **维度**：时间、主题、标签、内容
- **模式**：精确查找 / 探索发现 / 关联搜索 / 聚合整理

### 2.2 实体类型与发散维度

| 实体类型 | 示例 | 发散维度 |
|---------|------|---------|
| **人** | 张三、李经理 | 联系记录、沟通历史、共同参与的项目 |
| **项目** | 智能搜索Skill | 项目目标、里程碑、相关资源、参与人员 |
| **时间** | 上周、3月份、2024年Q1 | 该时间段内的所有笔记、周期性模式 |
| **主题** | AI、架构设计 | 相关概念、上下游话题、实现方案 |
| **任务** | 待办、会议、决策 | 任务状态、负责人、截止时间 |

### 2.3 搜索模式判断

| 模式 | 特征 | 处理策略 |
|------|------|---------|
| **精确查找** | 有明确时间/主题 | 直接搜索，快速收敛 |
| **探索发现** | 模糊表达如"看看"、"了解一下" | 宽范围搜索，提供概览 |
| **关联搜索** | "相关的"、"和...有关" | 先找锚点，再扩散关联 |
| **聚合整理** | "整理"、"汇总"、"所有" | 全面搜索，分类聚合 |

## 3. 策略执行层

### 3.1 四层工具调用链

```
L1: search_notes ───────→ 快速筛选候选笔记
            ↓
L2: get_note_outline ───→ 了解笔记结构，定位相关章节
            ↓
L3: search_note_content ─→ 在候选笔记中精确搜索
            ↓
L4: read_blocks ────────→ 精确定位并读取目标内容
```

### 3.2 何时使用哪一层

| 层级 | 适用场景 | 返回信息 |
|------|---------|---------|
| **L1** | 初始筛选、快速定位 | 笔记ID、标题、更新时间 |
| **L2** | 了解笔记结构、判断相关性 | 大纲、段落标题、block类型 |
| **L3** | 确认内容匹配、提取片段 | 匹配段落、关键词位置 |
| **L4** | 获取完整内容、深度分析 | 完整文本、图片、表格 |

### 3.3 典型调用序列

**场景A：精确查找特定笔记**
```
search_notes(keyword="项目规划")
    → get_note_outline(note_id) 确认结构
    → read_blocks(block_id) 读取目标段落
```

**场景B：探索性浏览**
```
search_notes(keyword="AI", limit=20)
    → 对前5个结果 get_note_outline 了解结构
    → search_note_content 在相关笔记中深度搜索
    → read_blocks 读取关键内容
```

**场景C：关联搜索**
```
get_note_outline(current_note_id) 获取当前笔记主题
    → search_notes(keyword=提取的主题词)
    → search_note_content 在结果中搜索关联内容
    → read_blocks 读取相关段落
```

## 4. 资产沉淀层

### 4.1 沉淀内容

每次搜索完成后，保存到本地JSON：

```json
{
  "timestamp": "2024-03-15T10:30:00Z",
  "original_query": "找一下上周关于AI的笔记",
  "extracted_entities": {
    "time": "last_week",
    "topic": "AI"
  },
  "search_path": [
    {"tool": "search_notes", "params": {...}, "results_count": 15},
    {"tool": "get_note_outline", "note_id": "..."},
    {"tool": "read_blocks", "block_id": "..."}
  ],
  "final_results": ["note-id-1", "note-id-2"],
  "effectiveness": "high"
}
```

### 4.2 沉淀用途

- **模式识别**：发现高频搜索主题
- **关联建立**：记录经常被一起搜索的主题
- **路径优化**：积累高效的搜索路径

## 5. 发散搜索逻辑

### 5.1 人 → 发散维度

当识别到人名时，搜索：
1. 直接提及该人的笔记
2. 该人参与的会议记录
3. 与该人相关的项目笔记
4. 发送给/来自该人的任务/决策

### 5.2 项目 → 发散维度

当识别到项目名称时，搜索：
1. 项目立项、规划文档
2. 项目里程碑、进度记录
3. 项目相关的技术方案
4. 项目会议、决策记录
5. 项目资源、参考材料

### 5.3 主题 → 发散维度

当识别到技术/业务主题时，搜索：
1. 该主题的直接相关内容
2. 相关概念（如"AI"→"机器学习"、"深度学习"）
3. 实现方案、最佳实践
4. 相关问题、解决方案

---

## 6. 结果融合原则

### 6.1 多维度结果融合

当使用多个维度搜索时：
1. **交集优先**：多维度都匹配的笔记排名靠前
2. **时效性加权**：近期笔记适当提升排名
3. **多样性保证**：避免同一笔记的多个段落垄断结果

### 6.2 结果排序因素

| 因素 | 权重 | 说明 |
|------|------|------|
| 匹配维度数 | 高 | 多维度匹配 = 高相关性 |
| 内容新鲜度 | 中 | 近期笔记优先 |
| 笔记完整性 | 低 | 结构完整的笔记优先 |

---

## 7. 错误处理与降级

### 7.1 工具调用失败

| 失败场景 | 处理策略 |
|---------|---------|
| search_notes 返回空 | 扩大关键词范围、尝试同义词 |
| get_note_outline 失败 | 跳过该笔记，继续处理其他结果 |
| read_blocks 失败 | 尝试读取相邻block或跳过 |

### 7.2 超时处理

- 单个工具调用超时：记录失败，继续其他调用
- 整体搜索超时：返回已获取的部分结果，告知用户结果不完整

## 8. 与上层Skill的协作

本Skill作为基础能力，为上层Skill提供：

```
上层Skill (如：笔记日报生成、项目知识库构建)
            ↓ 调用
wps-note-intelligent-search
            ↓ 使用 MCP 工具
wps-note-search MCP
```

上层Skill只需提供自然语言查询，本Skill负责：
1. 解析查询意图
2. 规划搜索策略
3. 执行工具调用链
4. 融合并返回结果
