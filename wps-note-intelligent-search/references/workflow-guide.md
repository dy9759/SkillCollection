# WPS Note 智能搜索 - 工作流示例

> 详细的工作流示例，展示 Claude 如何处理不同类型的搜索请求

---

## 示例1：精确查找特定笔记

### 用户请求
> "找一下上周关于项目规划的会议纪要"

### 工作流步骤

**Step 1: 意图解析**

提取的实体：
- 时间：上周（相对时间，需计算具体日期范围）
- 主题：项目规划
- 类型：会议纪要

判断搜索模式：精确查找（有明确时间和主题）

**Step 2: 策略规划**

搜索维度优先级：
1. 内容维度：关键词"项目规划"、"会议纪要"
2. 时间维度：上周创建的笔记
3. 标题维度：包含"会议"、"纪要"

**Step 3: 工具调用链**

```
# L1: 快速筛选
search_notes(
  keyword="项目规划",
  since="2024-03-01T00:00:00Z",
  until="2024-03-07T23:59:59Z"
)
→ 返回 5 个候选笔记

# L2: 了解结构
对前3个结果调用 get_note_outline(note_id)
→ 确认哪个是会议纪要

# L3: 精确搜索（如需要）
search_note_content(
  note_id="目标笔记ID",
  keyword="项目规划"
)
→ 确认内容匹配

# L4: 读取内容（如需要）
read_blocks(block_id="会议决议段落")
→ 获取具体决议内容
```

**Step 4: 结果整理**

返回：
- 笔记标题："项目规划会议纪要 - 2024.3.5"
- 创建时间：2024-03-05
- 相关片段：会议决议摘要
- 匹配维度：["content", "time", "title"]

## 示例2：探索性浏览

### 用户请求
> "看看我最近关于AI的笔记"

### 工作流步骤

**Step 1: 意图解析**

提取的实体：
- 时间：最近（默认过去30天）
- 主题：AI

判断搜索模式：探索发现（模糊表达"看看"）

**Step 2: 策略规划**

搜索策略：
1. 宽范围搜索：关键词"AI"、"人工智能"、"机器学习"
2. 获取概览：了解有哪些类型的AI相关内容
3. 分类呈现：按主题或时间组织结果

**Step 3: 工具调用链**

```
# L1: 宽范围搜索
search_notes(keyword="AI", limit=20)
→ 返回 15 个结果

search_notes(keyword="人工智能", limit=20)
→ 返回 8 个结果（部分与上面重复）

# 合并去重，得到 18 个唯一笔记

# L2: 了解结构（抽样）
对前5个结果调用 get_note_outline
→ 了解这些笔记的类型和结构

# 分类：
# - 技术方案：3篇
# - 学习笔记：2篇
# - 会议讨论：1篇

# L3: 深度搜索（可选）
如果用户需要更多细节：
search_note_content(note_id, keyword="AI")
→ 在特定笔记中搜索相关内容
```

**Step 4: 结果整理**

返回概览：
```
找到 18 篇关于AI的笔记（最近30天）：

【技术方案】3篇
- AI搜索架构设计
- 智能推荐算法方案
- 大模型接入设计

【学习笔记】2篇
- Transformer模型学习
- Prompt Engineering实践

【会议讨论】1篇
- AI功能评审会议

需要我详细介绍哪一类内容？
```

## 示例3：关联搜索

### 用户请求
> "和当前这个项目相关的笔记有哪些"

### 工作流步骤

**Step 1: 意图解析**

提取的实体：
- 锚点：当前笔记（从context获取）
- 关系：相关

判断搜索模式：关联搜索（"相关的"关键词）

**Step 2: 策略规划**

关联维度：
1. 主题关联：相同关键词、标签
2. 时间关联：同一时间段创建/修改
3. 显式关联：笔记中的链接引用

**Step 3: 工具调用链**

```
# 获取当前笔记信息
get_note_outline(note_id="current-note-id")
→ 获取标题、标签、结构

# 提取主题关键词："智能搜索"、"Skill开发"

# L1: 主题关联搜索
search_notes(keyword="智能搜索")
→ 返回 10 个结果

search_notes(keyword="Skill开发")
→ 返回 6 个结果

# L2: 标签关联搜索（如有标签）
search_notes(tags=["项目", "开发"])
→ 返回 8 个结果

# L3: 在候选结果中深度搜索
对最相关的3个笔记：
search_note_content(note_id, keyword="智能搜索")
→ 确认内容关联度

# L4: 读取关键内容
read_blocks(block_id="相关段落")
→ 提取关联的具体内容
```

**Step 4: 结果整理**

返回：
```
与"智能搜索Skill开发"相关的笔记：

【强相关】
- 智能搜索架构设计
- Skill开发规范
- MCP工具使用指南

【可能相关】
- 项目进度周报（提及此项目）
- 技术分享：AI应用实践

【同一时期创建】
- 会议纪要：3月技术规划
```

## 示例4：发散搜索（以人为中心）

### 用户请求
> "找一下张三相关的所有信息"

### 工作流步骤

**Step 1: 意图解析**

提取的实体：
- 人：张三

发散维度：
- 直接提及张三的笔记
- 张三参与的会议
- 与张三相关的项目
- 发送给张三的任务

**Step 2: 策略规划**

多维度发散搜索：
1. 直接搜索：关键词"张三"
2. 会议搜索：关键词"张三" + "会议"
3. 项目搜索：在项目中搜索"张三"
4. 任务搜索：关键词"张三" + "待办/任务"

**Step 3: 工具调用链**

```
# 维度1：直接提及
search_notes(keyword="张三")
→ 返回 12 个结果

# 维度2：会议记录
search_notes(keyword="张三 会议")
→ 返回 5 个结果

# 维度3：任务相关
search_notes(keyword="张三 待办")
→ 返回 3 个结果

# 合并去重，得到 15 个唯一笔记

# L2: 了解结构
对包含"会议"的笔记调用 get_note_outline
→ 确认会议类型和角色

# L3: 精确搜索
search_note_content(note_id, keyword="张三")
→ 找到具体提及位置

# L4: 读取相关内容
read_blocks(block_id="包含张三的段落")
→ 获取上下文
```

**Step 4: 结果整理**

按类别组织：
```
张三相关信息汇总：

【会议参与】5次
- 3月5日：项目启动会（参与者）
- 3月12日：技术评审会（主讲人）
...

【负责的任务】3项
- 完成架构设计文档
- 评审API接口规范
...

【相关项目】
- 智能搜索Skill（核心成员）
- 知识库重构项目（参与者）
```

## 示例5：聚合整理

### 用户请求
> "整理一下我所有的待办事项"

### 工作流步骤

**Step 1: 意图解析**

提取的实体：
- 任务类型：待办事项
- 范围：所有（全面搜索）

判断搜索模式：聚合整理（"整理"、"所有"）

**Step 2: 策略规划**

全面搜索策略：
1. 搜索关键词："待办"、"TODO"、"任务"
2. 深度搜索：在所有笔记中搜索
3. 提取任务：从匹配内容中提取具体任务项
4. 分类聚合：按状态、优先级、截止时间组织

**Step 3: 工具调用链**

```
# L1: 全面搜索（使用deep模式）
search_notes(keyword="待办", limit=100)
→ 返回 25 个结果

search_notes(keyword="TODO", limit=100)
→ 返回 10 个结果

search_notes(keyword="任务", limit=100)
→ 返回 30 个结果

# 合并去重

# L2: 了解结构
对前10个最相关的笔记调用 get_note_outline
→ 识别包含任务列表的笔记

# L3: 深度搜索任务项
对候选笔记：
search_note_content(note_id, keyword="待办")
→ 找到具体任务段落

# L4: 读取任务详情
read_blocks(block_id="任务列表段落")
→ 提取任务描述、状态、截止时间
```

**Step 4: 结果整理**

聚合输出：

【中优先级】8项
□ 更新项目文档
□ 整理会议纪要
...

【低优先级】4项
□ 学习新技术
...

【已完成待确认】2项
□ 提交代码审查

## 工具调用最佳实践

### 1. 分层递进

总是从L1开始，根据需要逐步深入：
- 不要一上来就调用 read_blocks
- 先用 search_notes 缩小范围
- 用 get_note_outline 了解结构
- 最后才用 read_blocks 读取具体内容

### 2. 并行调用

独立的工具调用可以并行：
```
# 可以并行
search_notes(keyword="A")
search_notes(keyword="B")

# 需要串行
search_notes() → get_note_outline() → read_blocks()
```

### 3. 结果缓存

在同一轮对话中：
- 已获取的 outline 可以复用
- 已读取的 blocks 不需要重复读取

### 4. 超时控制

- 单个工具调用设置合理超时（5-10秒）
- 整体搜索流程设置总超时（30-60秒）
- 超时后返回已获取的部分结果
