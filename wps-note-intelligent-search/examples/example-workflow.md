# 使用示例：智能搜索工作流

## 示例 1: 销售查找客户信息

**用户输入:**
```
帮我找一下张总的信息
```

**意图解析层处理:**
- 识别实体: "张总" (人物)
- 意图类型: find (查找)
- 发散维度: [联系方式, 近期沟通, 参与项目, 相关团队, 个人偏好]

**策略执行层处理:**
1. 搜索包含"张总"的笔记
2. 搜索标签含"客户"或"张总"的笔记
3. 搜索会议记录中提及"张总"的内容
4. 聚合结果并按时间排序

**输出结果:**
```json
{
  "results": [
    {
      "note_title": "张总 - ABC项目沟通纪要",
      "relevance": 0.95,
      "matched_dimensions": ["人物", "会议记录"]
    },
    {
      "note_title": "客户联系方式汇总",
      "relevance": 0.88,
      "matched_dimensions": ["人物", "联系方式"]
    }
  ],
  "divergent_suggestions": [
    "查看张总参与的 ABC项目 详情",
    "查看与张总相关的 李经理 信息",
    "查看近期与张总的 会议记录"
  ]
}
```

**资产沉淀:**
- 记录搜索模式: "{人物}的信息" → 人物+关联信息
- 建立实体关系: 张总 ←→ ABC项目, 张总 ←→ 李经理

## 示例 2: 产品经理查找项目文档

**用户输入:**
```
Q3规划相关的所有内容
```

**意图解析层处理:**
- 识别实体: "Q3规划" (项目/规划)
- 意图类型: discover (发现关联)
- 发散维度: [目标, 里程碑, 资源需求, 依赖项目, 负责人, 进展状态]

**策略执行层处理:**
1. 搜索标题含"Q3规划"的笔记
2. 搜索标签含"Q3"或"规划"的笔记
3. 搜索内容提及"Q3目标"、"Q3里程碑"的笔记
4. 按文档类型分组（会议纪要/计划文档/进展更新）

**输出结果:**
```json
{
  "results": [
    {
      "note_title": "Q3产品规划 v2.0",
      "relevance": 1.0,
      "matched_dimensions": ["项目名"]
    },
    {
      "note_title": "Q3里程碑跟踪",
      "relevance": 0.92,
      "matched_dimensions": ["里程碑", "时间关联"]
    },
    {
      "note_title": "7月迭代复盘",
      "relevance": 0.75,
      "matched_dimensions": ["时间关联", "进展状态"]
    }
  ],
  "divergent_suggestions": [
    "查看 Q3规划 的 里程碑 完成情况",
    "查看负责 Q3规划 的 相关人员",
    "查看 Q3规划 依赖的 其他项目"
  ]
}
```

## 示例 3: 开发者查找技术方案

**用户输入:**
```
前端性能优化有什么方案
```

**意图解析层处理:**
- 识别实体: "前端性能优化" (技术主题)
- 意图类型: summarize (总结方案)
- 发散维度: [技术方案, 实施记录, 测试数据, 相关会议, 最佳实践]

**策略执行层处理:**
1. 搜索标题含"性能优化"的笔记
2. 搜索代码块中含优化方案的笔记
3. 搜索标签含"前端"、"性能"的笔记
4. 提取方案概要，去重归类

**输出结果:**
```json
{
  "results": [
    {
      "note_title": "前端性能优化实践总结",
      "relevance": 0.95,
      "matched_dimensions": ["主题匹配", "技术方案"],
      "extracted_summary": "懒加载、代码分割、缓存策略..."
    },
    {
      "note_title": "首页加载优化方案评审",
      "relevance": 0.82,
      "matched_dimensions": ["会议记录", "技术方案"]
    }
  ],
  "divergent_suggestions": [
    "查看 前端性能优化 的 实施记录",
    "查看相关的 测试数据",
    "查看 性能优化 的 最佳实践案例"
  ]
}
```

## 示例 4: 搜索资产复用

**场景:** 用户上周搜索过"张总"，本周搜索"ABC项目"

**资产沉淀层处理:**
1. 发现历史搜索记录中有"张总"
2. 发现实体关系: 张总 ←participate→ ABC项目
3. 推荐关联: "您之前查看过 张总，他是 ABC项目 的参与者"

**用户价值:**
- 无需重复搜索人物信息
- 自动建立项目与人员的关联
- 形成个人知识图谱
