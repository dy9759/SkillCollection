# 知识库检索策略

用户接触到新闻后，从新闻中提取 3-5 个核心关键词（实体词、核心概念、公司/人物/技术名），在用户的**整个笔记库**中检索关联内容。

## 检索策略

按优先级执行，取并集去重：

### 1. 按关键词全文搜索（最重要）

```
search_notes(keyword=关键词1)   — 搜索用户所有笔记
search_notes(keyword=关键词2)   — 补充搜索
search_notes(keyword=关键词3)   — 补充搜索
```

关键词要覆盖不同维度：公司名、技术名、行业名、人物名。

### 2. 按标签搜索相关新闻

```
search_notes(tags=["新闻//AI"])   — 按领域标签找同领域的历史新闻
```

补充时间线和行业背景。

### 3. 按时间范围搜索近期笔记

```
search_notes(since=近一个月)   — 找用户近期关注的内容
```

捕捉用户当前的关注点和工作上下文。

### 4. 扩展搜索

若结果不足，尝试更宽泛的同义词或上位概念搜索。

## 搜索范围

**不限于 `#新闻` 标签的笔记**，以下类型都是有价值的关联源：

- 工作笔记
- 会议记录
- 项目文档
- 学习笔记
- 读书笔记
- 过往新闻剪报

示例：一条关于 OpenAI 融资的新闻，可能和用户的"AI 战略规划"工作笔记、"技术选型会议"记录、"竞品分析"文档都有关联。

## 结果处理

- 检索结果上限 10-15 篇，排除当前新闻自身。
- 对检索到的关联笔记：
  - 调用 `get_note_outline(note_id)` 获取结构预览，判断相关性。
  - 对高相关笔记调用 `read_note(note_id)` 或 `read_section` 读取关键内容。
  - 最多深度读取 5 篇关联笔记，避免过多 token 开销。
  - 提取每篇关联笔记的核心观点、数据、时间线。
  - 特别注意用户的**个人观点、决策、计划**——这些是产出个性化 insight 的关键素材。
