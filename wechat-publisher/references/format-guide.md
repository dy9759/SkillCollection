# 格式规范指南

本文档详细说明 wechat-channel-formator 的格式规范和使用方法。

---

## 占位符标签速查表

在 WPS 笔记中使用以下占位符标签控制导出样式：

| 标签 | 语法示例 | Markdown 输出 | HTML 效果 |
|------|----------|---------------|-----------|
| `<b/>` | `<b/>重要内容</b>` | `!!重要内容!!` | 橙色加粗整段 |
| `<h2/>` | `<h2/>章节标题</h2>` | `## 章节标题` | 蓝色大标题（一级章节） |
| `<h3/>` | `<h3/>小节标题</h3>` | `### 小节标题` | 蓝色小标题（二级分节） |
| `<bq/>` | `<bq/>引用内容</bq>` | `> 引用内容` | 灰底蓝边引用块 |
| `<note/>` | `<note/>注释内容</note>` | `~~注释内容~~` | 灰色小字注释 |

### 标签详细说明

#### `<b/>` - 橙色加粗

用于强调重要内容，整段文字将显示为橙色加粗。

```xml
<b/>这是整段橙色加粗的文字，适合核心论点或金句。</b>
```

效果：
- Markdown: `!!这是整段橙色加粗的文字，适合核心论点或金句。!!`
- HTML: 橙色加粗段落

#### `<h2/>` - 蓝色大标题

用于一级章节分隔，显示为蓝色大标题。

```xml
<h2/>01｜基础知识</h2>
```

效果：
- Markdown: `## 01｜基础知识`
- HTML: 蓝色大标题（18px，上边距 30px）

#### `<h3/>` - 蓝色小标题

用于章节内二级分节，显示为蓝色小标题。

```xml
<h3/>核心设计原则</h3>
```

效果：
- Markdown: `### 核心设计原则`
- HTML: 蓝色小标题（16px，上边距 20px）

#### `<bq/>` - 引用块

用于摘录观点、名言或补充说明。

```xml
<bq/>这是一个引用块的内容，适合摘录重要观点或补充说明。</bq>
```

效果：
- Markdown: `> 这是一个引用块的内容，适合摘录重要观点或补充说明。`
- HTML: 灰底蓝边引用块

#### `<note/>` - 注释小字

用于来源标注或补充说明，显示为灰色小字。

```xml
<note/>注：本文仅代表个人观点，具体产品选择请根据自身需求判断。</note>
```

效果：
- Markdown: `~~注：本文仅代表个人观点，具体产品选择请根据自身需求判断。~~`
- HTML: 灰色小字（14px）

## Markdown 扩展语法

### 块级元素

#### 普通段落

直接写正文，不加任何前缀。渲染为灰色正文（`rgb(100,106,115)`）。

```markdown
这是普通正文段落，可以用 **加粗词句** 强调重点。
```

#### 章节大标题 `##`

蓝色加粗，用于一级章节分隔。

```markdown
## 01｜基础知识
```

#### 小节标题 `###`

蓝色加粗，用于章节内二级分节（比 `##` 视觉权重低一级）。

```markdown
### 核心设计原则
```

> 注意：`##` 和 `###` 都是蓝色，不是橙色。

#### 橙色加粗整段 `!!...!!`

整段橙色加粗，用于核心结论或行动号召。**必须独占一行**，首尾各两个感叹号。

```markdown
!!用 Skill 一次性教会 Claude，之后每次都能直接用。!!
```

#### 无序列表 `- `

橙色圆点列表。

```markdown
- SKILL.md（必须）：带有 YAML 前置元数据的 Markdown 格式指令
- scripts/（可选）：可执行代码
- references/（可选）：按需加载的参考文档
```

#### 有序列表 `1. `

橙色序号列表。

```markdown
1. 第一步：克隆仓库
2. 第二步：安装依赖
3. 第三步：运行脚本
```

#### 引用块 `> `

灰底蓝边引用块。**连续多行 `>` 自动合并为一个块**，内部用 `<br>` 换行也可以。

```markdown
> 译者注：Level 1 元数据大约只消耗 ~100 tokens。<br>Level 2 的 SKILL.md 正文建议控制在 5,000 tokens 以内。
```

或多行写法（自动合并）：

```markdown
> 第一行内容
> 第二行内容
> 第三行内容
```

#### 代码块（有语言标注）

浅灰底白框 + 语法高亮 + 右上角语言标签。支持：`python`、`javascript`/`js`、`typescript`/`ts`、`bash`/`sh`、`yaml`、`json`、`css`、`html`。

````markdown
```python
def hello():
    print("Hello, world!")
```
````

#### 代码块（无语言标注）

灰底框 + 普通字体，用于结构化说明文本、示例格式等非代码内容。

````markdown
```
用例：项目 Sprint 规划
触发条件：用户说「帮我规划这个 Sprint」
结果：Sprint 完整规划，任务已创建
```
````

#### 分隔线 `---`

用于章节间的视觉分隔。

```markdown
---
```

#### 注释小字 `~~...~~`（独立行）

小号灰色注释文字。**必须独占一行**，整行被 `~~` 包裹。

```markdown
~~注：本文仅代表个人观点，具体产品选择请根据自身需求判断。~~
```

---

### 行内元素

#### 黑色加粗 `**...**`（行内）

用于段落内个别重要词句，渲染为**橙色加粗**。不要整段加粗。

```markdown
这是针对自己需求定制 Claude **最直接**的方式之一。
```

#### 整行加粗 `**整行**`

整行被 `**` 包裹时，渲染为独立橙色加粗段落（同 `!!...!!` 效果）。

```markdown
**「你觉得现在用哪个模型最好？」**
```

#### 行内代码 `` `code` ``

灰底深灰字等宽字体，用于模型名、参数名、命令等。

```markdown
以 `qwen3.5-plus` 为基准，`deepseek-v3.2` 反而贵 47.7%。
```

#### MD 链接 `[文字](url)`

渲染为蓝色可点击链接。

```markdown
平台地址：[xsct.ai](https://xsct.ai)
```

## 模板使用说明

### 模板结构

模板由 YAML 配置文件定义，包含以下部分：

```yaml
colors:      # 颜色配置
fonts:       # 字体配置
spacing:     # 间距配置
layout:      # 布局配置
features:    # 功能开关
```

### 创建自定义模板

1. 复制 `templates/default.yaml` 为新文件（如 `templates/custom.yaml`）
2. 修改配置项
3. 使用时指定 `--template "custom"`

### 模板变量

HTML 模板中可使用以下变量：

| 变量 | 说明 |
|------|------|
| `{{ content }}` | 文章主体内容 |
| `{{ title }}` | 文章标题 |
| `{{ date }}` | 导出日期（YYYY-MM-DD） |

## 排版风格规范

1. **标题不加句号**
2. `##` 标题建议加章节序号，如 `## 01｜基础知识`
3. 引用块用于**补充说明、译者注、重要提示**，不用于普通正文
4. `!!橙色加粗!!` 用于**全文最重要的 1-3 句话**，不要滥用
5. 列表项末尾不加句号（除非是完整句子）
6. 代码块必须标注语言（`python`、`bash`、`yaml` 等）；说明性文本用无语言代码块

## 完整示例

在 WPS 笔记中编写：

```xml
<h2/>01｜Skill 是什么？

Skill 是以文件夹形式打包的一组指令，用来教会 Claude 怎么处理特定任务或工作流。这是针对自己需求定制 Claude <b/>最直接</b>的方式之一。

每次对话都要重新解释偏好和流程太低效了——<b/>用 Skill 一次性教会 Claude，之后每次都能直接用。</b>

<h3/>Skill 的文件结构

一个 Skill 文件夹包含：

- `SKILL.md`（必须）：带有 YAML 前置元数据的指令文件
- `scripts/`（可选）：可执行代码
- `references/`（可选）：按需加载的参考文档

<h3/>三级加载机制

<bq/>Level 1（YAML 元数据）始终加载，约消耗 ~100 tokens。<br>Level 2（SKILL.md 正文）相关时才加载，建议控制在 5,000 tokens 以内。<br>Level 3（链接文件）按需读取，不读不占 token。</bq>

---

<h2/>快速上手

```bash
git clone https://github.com/example/skill-creator
cd skill-creator
python3 scripts/init_skill.py my-skill
```

更多文档见 [ClawHub](https://clawhub.ai)。

<note/>注：skill-creator 工具本身也是一个 Skill，可以用来构建其他 Skill。</note>
```

## 转换命令

导出笔记：

```bash
python3 scripts/export-to-html.py --note-id "笔记ID" --template default
```

输出：同目录下 `{标题}_formatted.html`，直接粘贴到公众号编辑器即可。

---

## 注意事项

1. **行内代码限制**: WPS 笔记 XML 不支持 `<code>` 标签，行内代码会被转换为纯文本
2. **引用块约束**: MCP XML 对引用块有严格的内容限制
3. **内容长度**: 超过 2000 字符的内容需要分批处理
4. **图片处理**: 本地图片会自动转为 base64 内嵌，网络图片直接引用
